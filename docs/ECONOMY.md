# 局内分数 ↔ 局外金币（经济与结算）

实现位置：[`game.js`](../game.js)（`computeRunScore`、`awardKillScore`、`triggerGameOver`、常量 `COIN_CONVERSION_RATIO`）。

---

## 一句话

- **局内**会累计「击杀分」`state.score`，游戏结束时再和波次、时间、等级拼成 **终局总分** `finalScore`。
- **局外金币**只在 **死亡/结算** 时增加：`金币 = 终局总分 ÷ 20`（**向下取整**）。
- 比例可记：**每 20 分终局分 ≈ 1 金币**（整除，零头丢弃）。

### 和直觉一致吗：活得越久、打得越好 → 币越多？

**是的，逻辑上已经这么设计了。**

- **终局总分**里同时包含：**击杀分**（打得猛）、**波次**（推进得远）、**生存秒数**（活得久）、**等级**（成长）。一般打得越好、活得越久，这几项都会更高，总分就更高。
- **金币**与终局总分是**线性关系**：`金币 = ⌊终局总分 / 比率⌋`。所以在同一套公式下，**同一玩家、更好的一局 → 终局分更高 → 结算金币更多**；调大 `COIN_CONVERSION_RATIO` 只是**全体少发一点币**，并不会改成「高分反而少拿」。
- 若希望**更强调存活时间**，可以以后单独加大 `elapsed` 的系数，或弱化其它项；那是数值调参，不是当前「转化率」这一条能单独表达的。

## 终局总分怎么算

游戏结束时：

```text
终局总分 = ⌊ 击杀分 + 波次×220 + 生存秒数×2.5 + 等级×45 ⌋
```

| 部分 | 含义 | 系数 |
|------|------|------|
| 击杀分 | 本局所有击杀累加（见下节） | — |
| 波次 | 当前 `state.wave` | **+220 / 波** |
| 时间 | 本局 `state.elapsed`（秒） | **+2.5 / 秒** |
| 等级 | 结束时玩家等级 | **+45 / 级** |

`⌊·⌋` 表示最后向下取整为整数（与代码中 `Math.floor` 一致）。

**排行榜**用的也是同一个「终局总分」（`computeRunScore()`），与金币共用这一数值。

## 击杀分怎么涨（局内 `state.score`）

每次击杀：

1. **基础分**  
   - **BOSS**：固定 **520**  
   - **非 BOSS**：`round(18 + 该敌人的 xpValue × 2.2)`

2. **连击倍率**（在约 **2.4 秒** 内继续击杀会维持/推高连击，超时重置）

   ```text
   倍率 = min(4, 1 + tier×0.35 + withinTier×0.08)
   ```

   - `tier = ⌊(连击数−1) / 3⌋`（每 3 连进入更高一档）  
   - `withinTier = (连击数−1) mod 3`  
   - 连击数指「当前连击计数」`state.combo.count`（每次击杀 +1）

3. **本次击杀得分**  
   `round(基础分 × 倍率)`，并加入 `state.score`。

## 局内 → 局外：金币转化率

```text
本局获得金币 = max(0, ⌊ 终局总分 / 20 ⌋)
```

| 概念 | 数值 |
|------|------|
| 代码常量 | `COIN_CONVERSION_RATIO = 20` |
| 通俗理解 | **20 点终局分换 1 金币**（整数除法，余数不要） |
| 例 | 终局总分 400 → 金币 **20**；265 → **13** |

调大该常数会**降低**「分数→金币」的慷慨程度（单局金币变少）；调小则变慷慨。

金币写入 `state.meta.coins` 并持久化（`localStorage`），仅在结算时增加这一笔。

## 局外金币还从哪来 / 花到哪（备忘）

与「终局分→币」并列的**其它经济**，便于对照：

| 场景 | 规则（代码侧） |
|------|----------------|
| 幸运转盘 | 每次旋转消耗 **80** 金币；奖品含固定额度金币或宠物等（见 `WHEEL_SPIN_COST` 与转盘配置） |
| 重复宠物折算 | 约为该宠物**商店价格 × 0.6** 的金币返还（`Math.floor`） |
| 宠物商店 | 各宠物标价在 `petDefinitions`（如 320～1500） |

若调整 `COIN_CONVERSION_RATIO` 或 `computeRunScore` 公式，请同步更新本文与 [`README.md`](./README.md) 中的维护说明。
